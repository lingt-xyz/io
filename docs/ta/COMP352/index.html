Traversals#

Question#

info

Problem : Draw a tree TTT given that:

  • A pre-order traversal of TTT yields: E K D M J G I A C F H B LE\ K\ D\ M\ J\ G\ I\ A\ C\ F\ H\ B\ LE K D M J G I A C F H B L.
  • A post-order traversal of TTT yields: D J I G A M K F L B H C ED\ J\ I\ G\ A\ M\ K\ F\ L\ B\ H\ C\ ED J I G A M K F L B H C E.
tip

Use divide and conquer approach.

  1. pre-orderyields E K D M⋯⇒EE\ K\ D\ M \cdots\Rightarrow EE K D M⋯⇒E is the root of TTT, KKK is the first child of TTT.
  2. post-order yields⋯B H C E⇒E\cdots B\ H\ C\ E\Rightarrow E⋯B H C E⇒Eis the root of TTT, CCC is the last child of TTT.
  3. If EEE has another child which is in-between KKK and CCC, it should be DDD because of pre-order. However, DDD is the first node in post-order, soDDDcannot be in-between KKK and CCC.
  4. EEE only has a left child KKK and a right child C⇒C\RightarrowC⇒
    • From pre-oder, K D M J G I AK\ D\ M\ J\ G\ I\ AK D M J G I A is a left-sub-tree; C F H B LC\ F\ H\ B\ LC F H B L is a right-sub-tree.
    • From post-order, D J I G A M KD\ J\ I\ G\ A\ M\ KD J I G A M K is a left-sub-tree; F L B H CF\ L\ B\ H\ CF L B H C is a right-sub-tree.

caution

    • Pre-order traversal yields: K D M J G I AK\ D\ M\ J\ G\ I\ AK D M J G I A.

    • Pre-order traversal yields: C F H B LC\ F\ H\ B\ LC F H B L.
    • Post-order traversal yields: F L B H CF\ L\ B\ H\ CF L B H C.

Shortest paths#

Problem Define#

Given a weighted graph and two vertices uuu and vvv, we want to find a path of minimum total weight between uuu and vvv.

info

Types of shortest paths problems

  • Single-Source shortest-paths problem
  • Single-destination shortest-paths problem
  • Single-pair shortest-path problem
  • All-pairs shortest-paths problem

Shortest Path Properties#

  1. A sub-path of a shortest path is itself a shortest path.
  2. There is a tree of shortest paths from a start vertex to all the other vertices.

Dijkstra’s Algorithm#

Dijkstra’s algorithm solves single-source shortest-paths problem. It is a type of greedy algorithm.

Assumptions of Dijkstra's Algorithm#

  1. The graph is connected.
  2. The edges are undirected.
  3. The edge weights are non-negative.

Edge Relaxation#

Edge relaxation is a helper function.

info

Stretch it first, then relax, so it's called relaxation.

  1. Let D[u]D[u]D[u] denote the length of the best path from vvv to uuu that could have been found so far.
  2. Initially, D[v]=0D[v]=0D[v]=0, and D[u]=∞D[u]=\inftyD[u]=∞ for each other vertex uuu in the graph.
  3. D[u]D[u]D[u] will afterwards be updated if a better path is found that results in a smaller shortest path value to uuu. This step is relaxation.

Dijkstra's Algorithm#

Two sets#

  • CCC contains the vertices which we have already found the shortest paths to them.
    • After a vertex was put in CCC, we should never update it again, because the solution has been found at the moment it was put in.
  • SSS contains the vertices connecting to the vertices in CCC.

Steps#

  1. Initially, C=∅,S={v}C=\emptyset, S=\{v\}C=∅,S={v}, where vvv is the starting vertex.
  2. Do edge relaxation initialization; that is, D[v]=0D[v]=0D[v]=0 and D[u]=∞D[u]=\inftyD[u]=∞ for each other vertex uuu.
  3. Select the vertex sss in SSS which has the shortest path tovvv.
    1. Move sss to CCC.
    2. For every vertex which is connected to sss and not in CCC, do edge relaxation.
    3. Add these connected vertices to CCC.
  4. Repeating step #3 util S=∅S=\emptysetS=∅.
